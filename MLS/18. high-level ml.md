## Higher-Level AI/ML Services

AWS는 SageMaker처럼 모델을 직접 설계하지 않아도, **완전관리형 AI 서비스**를 제공합니다.  
이 서비스들은 공통적으로 **AutoML 기반**이며, **사전 학습된 모델**을 활용합니다.

대표 서비스:

- Amazon **Comprehend** (자연어 처리)
- Amazon **Translate** (번역)
- Amazon **Transcribe** (음성 → 텍스트)
- Amazon **Polly** (텍스트 → 음성)
- Amazon **Rekognition** (컴퓨터 비전)
- Amazon **Forecast** (시계열 예측)
- Amazon **Lex** (대화형 챗봇)
    

---

### Amazon Comprehend – 자연어 처리 (NLP)

#### 주요 기능

- **텍스트 분석(NLP) 서비스**
    
- 입력: 소셜 미디어 글, 이메일, 문서, 웹페이지, 의료 기록 등
    
- 기능 요약:
    
    - **핵심 구문(Key Phrases)** 추출
    - **엔터티(Entity)** 식별 (사람, 조직, 위치 등)
    - **감정(Sentiment)** 분석
    - **언어(Language)** 판별
    - **구문(Syntax)** 분석 (품사 등)
    - **주제(Topics)** 분류
    - **PII (개인식별정보)** 탐지 및 비식별화(Redaction)
    - **특정 대상에 대한 감정(Targeted Sentiment)** 분석
    - **자체 데이터로 모델 훈련(Custom Classifier)** 가능
        

#### 예시 기능별 설명

|기능|설명|
|---|---|
|**Entities**|“Amazon”, “New York” 같은 실체명 인식|
|**Key Phrases**|핵심 문구 추출 (예: “customer satisfaction”)|
|**Language**|입력 문장의 언어 자동 감지|
|**Sentiment**|문서의 전반적 감정(긍정/부정/중립) 분석|
|**Syntax**|문법 구조 및 품사 분석|

## Amazon Translate – 번역 서비스

#### 주요 기능

- **딥러닝 기반 실시간 번역 서비스**
    
- 다국어 자동 번역 지원
    
- **Custom Terminology** 지원
    
    - 기업 고유 명칭, 브랜드명 등 고유어를 지정 가능
    - CSV 또는 TMX 포맷으로 업로드
        

예:  
“Amazon Web Services” → “AWS” 로 번역하도록 지정 가능

### Amazon Transcribe – 음성 인식 서비스

#### 주요 기능

- **음성을 텍스트로 변환 (Speech-to-Text)**
    
- 입력 포맷: `FLAC`, `MP3`, `MP4`, `WAV`
    
- **스트리밍 오디오 지원** (`HTTP/2`, `WebSocket`)
    
- **지원 언어**: 영어, 프랑스어, 스페인어 등
    
- **자동 언어 감지 기능** 포함
    

#### 세부 기능

|기능|설명|
|---|---|
|**Speaker Identification**|발화자 구분 가능 (화자 수 지정)|
|**Channel Identification**|예: 두 통화 채널(고객/상담원)을 분리 인식|
|**Custom Vocabulary**|고유 용어, 약어 등을 인식하도록 단어 사전 등록|
|**Vocabulary Tables**|발음(`IPA`), 표기(`DisplayAs`), 유사음(`SoundsLike`) 추가 가능|

#### 주요 사용 사례

- **고객센터 분석(Call Analytics)**  
    → 고객 감정, 말속도, “해지” 등 특정 문구 감지
    
- **의료 음성 인식 (Transcribe Medical)**  
    → 의료 전문용어 학습, HIPAA 적격 서비스
    
- **실시간 자막 생성(Subtitling)**

## Amazon Polly – 음성 합성 (Text-to-Speech)

#### 주요 기능

- **신경망 기반 음성 합성 (Neural TTS)**
    
- 수십 개 언어, 다양한 음색 제공
    

#### 기능 요약

|기능|설명|
|---|---|
|**Lexicons**|특정 단어나 문구의 발음을 사용자 정의 (예: “W3C” → “World Wide Web Consortium”)|
|**SSML (Speech Synthesis Markup Language)**|발음, 강조, 속도, 피치, 정지, 속삭임 등 세밀한 조절|
|**Speech Marks**|오디오 내 단어·문장 시작·끝 타임스탬프 제공 → **리핑싱크(lip-sync)** 가능|

### Amazon Rekognition – 컴퓨터 비전 서비스

#### 주요 기능

- **이미지 및 영상 분석 서비스**
    
- 기능 요약:
    
    - **객체(Object) 및 장면(Scene)** 인식
        
    - **얼굴 인식 (Face Detection & Comparison)**
        
    - **유명인(Celebrity) 인식**
        
    - **텍스트 감지 (Text in Image)**
        
    - **영상 분석 (Video Analysis)**
        
    - **부적절한 콘텐츠 필터링 (Moderation)**
        

#### 세부사항 (The Nitty Gritty)

- 입력: **S3 이미지** 또는 **바이트 업로드**
    
- S3에 이미 저장된 이미지를 사용하는 것이 빠름
    
- 얼굴 인식 정확도는 **조명, 해상도, 각도**에 민감
    
- 영상은 **Kinesis Video Streams**를 통해 처리
    
    - H.264 인코딩, 5~30 FPS 권장
        
    - 해상도 > 프레임레이트 우선
        
- Lambda와 연동하여 이미지 업로드 시 자동 분석 가능
    

#### Custom Labels

- 소량의 라벨링된 이미지로 자체 모델 학습 가능
    
- 예: **NFL**이 팀 로고나 경기 소품(폼 핑거 등)을 식별하도록 학습

### Amazon Forecast – 시계열 예측 서비스

#### 주요 기능

- **완전관리형 예측 서비스 (Fully Managed)**
    
- **AutoML**로 시계열 데이터에 최적 모델 자동 선택
    
- 지원 알고리즘:
    - ARIMA, DeepAR+, ETS, NPTS, CNN-QR, Prophet
        
- 활용 예시:
    - 재고, 가격, 매출, 경제 지표, 자원 수요 등
        
- 입력 구조:  
    **Dataset Groups**, **Predictors**, **Forecasts**
    

#### 알고리즘 요약

|알고리즘|설명|
|---|---|
|**CNN-QR**|CNN 기반 Quantile Regression – 대규모 시계열에 적합|
|**DeepAR+**|RNN 기반 – 대규모 데이터셋, 미래 예측용|
|**Prophet**|비선형 추세 + 계절성 모델|
|**NPTS**|비모수 시계열 – 희소 데이터에 적합|
|**ARIMA**|단순 시계열 예측 (<100 시계열)|
|**ETS**|지수 평활법 – 단순 데이터셋용|

### Amazon Lex – 챗봇 서비스

#### 개요

- **Alexa의 핵심 엔진** 기반
    
- **자연어 기반 챗봇** 구축 서비스
    

#### 핵심 개념

|구성 요소|설명|
|---|---|
|**Intent**|사용자의 의도 (예: “피자 주문하기”)|
|**Utterance**|사용자의 실제 문장 (예: “페퍼로니 피자 주문하고 싶어요”)|
|**Slot**|Intent 수행에 필요한 세부정보 (예: 크기, 토핑, 배달 시간 등)|
|**Lambda 함수**|Intent 수행 로직 처리|
|**배포 대상**|AWS Mobile SDK, Slack, Facebook Messenger, Twilio 등|

---

### Lex Automated Chatbot Designer

#### 기능 개요

- **기존 대화 기록(transcripts)**을 입력하면  
    Lex가 **자동으로 챗봇 설계**를 생성함.
    
- 딥러닝과 NLP로 **중복 제거 및 의도 분리** 수행.
    
- **Intents**, **요청 문구**, **Slot 값**을 자동 추출.
    
- **Amazon Connect**의 상담 대화 기록과 통합 가능.
